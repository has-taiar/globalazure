Azure Durable Functions. Building Serverless and Stateful Solutions on Azure. – Paco De La Cruz

Azure Functions is a serverless platform for running nano-services on the cloud which has been around for some time. By definition, Azure Functions are for single-purpose services with short-lived executions. However, more recently, Microsoft have launched Azure Durable Functions, which allows you to write stateful functions and long-running code orchestrations with all the benefits of a serverless environment. In this talk, Paco will talk about the capabilities of this new service offering, potential use cases, and demo how to leverage Durable Functions with other Azure offerings to build serverless and stateful solutions.

First steps with Terraform on Azure - David O'Brien

So you started your cloud journey and quickly discovered that you need to leverage some sort of Infrastructure as Code strategy. You also realised that writing long files of JSON ARM templates is not what humans were ever intended to do (leave that to the robots) and just as quickly as you got here move on to the next thing, the thing everybody is talking about, Terraform. Let us make this session about what it means to select Terraform as your tool of choice for managing infrastructure on Azure. What are the good and the potentially bad things? You got rid of JSON now, but what did you trade it in for? Let's look at some typical ways to get started and obviously also deploy some infrastructure. This session is open source, all the code used will be available for everybody to keep.

Reduce dev and ops frictions with Azure DevOps (from an Ops perspective) – Vic Perdana

As an IT pro, you are incentivized to keep things running whereas your dev team loves to add more features. In the session, you'll be introduced to Azure DevOps, a powerful suite from MS which can facilitate the alignment between Dev and Ops. The demonstration will involve a fictitious company with an existing app in Azure using a traditional approach and transition it to the new world of Azure DevOps. In this session we will demonstrate how to: • Get started with Azure DevOps • Use your existing scripts and move them to build pipelines • Create release pipelines for your dev, test, and production • Bonus: How you can add some production stuff (such as backup, monitoring, change processes) to the production release
Disclaimer: I am an IT pro :) so I will focus more on the infrastructure/Ops side of things.

RBAC in Kubernetes with Azure AD – Emad Alashi

So there is more than one Namespace in your Kubernetes cluster, and you definitely want to give access to more than one person in your team to the cluster to be able to do different actions. In this session we will talk about RBAC (Role-Based Access Control) in Kubernetes and how we can utilise Azure AD to achieve better access control.

Deep Dive into Azure Functions –Justin Yoo

There are three pillars building a serverless API application - portability, testability and discoverability. Azure Functions is one of the most popular serverless offerings from Azure and it supports those three aspects. By achieving these, your Azure Function app will be written and deployed more effectively and efficiently. Throughout this session, audiences will learn 1) how to deploy Azure Function app through containers to achieve portability, 2) how to implement dependency injection for better testability, and 3) how to render Swagger UI for better discoverability.
During the session, I'm going to connect to a Raspberry PI to show how to deploy Azure Functions through containers.

The Boring Security Talk – Kieran Jacobsen

We all know that securing our applications is a necessity, but it can be incredibly boring. With time and budget constraints, we often focus on the more exciting security aspects and tools. In this talk, we'll be looking at some of the aspects to our application security that are often overlooked; the software we depend upon, CI/CD infrastructure, sending email and resolving DNS. Vulnerabilities here might not result in a newsworthy breach, yet they are still worth discussing and defending.

Azure Virtual Datacenter Inside Out – Janaka Rangama

Azure Virtual Datacenter is a design approach that allows you to make the most of the Azure cloud platform's capabilities while keeping your existing security and networking policies intact. In this demo rich session we will dissect the core Azure VDC concepts including identity, encryption, software-defined networking, and compliance and showcase how to infuse them into your Azure environment. If you a Cloud Architect who is planning your next data centre in Microsoft Azure, this session is just the right fit for you.

Building APIs Rapidly with Azure Functions – Lars Klint

APIs are what makes the Internet go around. Every SaaS, data driven and on-demand kitten business exposes their data over an external or internal API. Which means you need one too, right? Of course you do. Creating an API that conforms with OpenAPI standards, can scale up and out, can be extended easily, and can be easily maintained is the Nirvana of API architecture. This is exactly the promise of building an API with Azure Functions, and it is super fast too!
In this session, you will learn to quickly create a full blown API using Azure Functions, create a proxy to forward requests, adhere to OpenAPI standards, easily extend and more. At the end of the talk we will have built an API that is (almost) production ready straight away. We then compare this process to other API frameworks to get the real world picture.

Our conference is dedicated to providing a harassment-free conference experience for everyone, regardless of gender, gender identity and expression, age, sexual orientation, disability, physical appearance, body size, race, ethnicity, religion (or lack thereof), or technology choices. Harassment includes offensive verbal comments related to gender, gender identity and expression, age, sexual orientation, disability, physical appearance, body size, race, ethnicity, religion, technology choices, sexual images in public spaces, deliberate intimidation, stalking, following, harassing photography or recording, sustained disruption of talks or other events, inappropriate physical contact, and unwelcome sexual attention. Conference staff will be happy to help participants contact hotel/venue security or local law enforcement, provide escorts, or otherwise assist those experiencing harassment to feel safe for the duration of the conference. We value your attendance. We expect participants to follow these rules at conference and workshop venues and conference-related social events.
